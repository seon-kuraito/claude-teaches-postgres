# 進度追蹤規則

---

## 自動更新時機

| 角色 | 觸發時機 | 更新內容 |
|------|---------|---------|
| `/planner` | 完成大綱規劃 | 建立 `學習進度.md` 初始版本 |
| `/teacher` | 完成章節教學 | 標記該章節「教學 ✅」，更新完成日期 |
| `/grader` | 完成測驗批改 | 更新測驗狀態、成績、需複習清單 |

---

## 更新方式

### 必須使用 Edit tool
- **禁止**整個檔案重寫（會導致其他章節資料遺失）
- **必須**使用 Edit tool 精確修改對應欄位
- 只更新需要變動的部分

### `/teacher` 更新範例

```markdown
# 更新前
| 01 | 認識資料庫 | - | - | - | - |

# 使用 Edit tool 更新
old_string: "| 01 | 認識資料庫 | - | - | - | - |"
new_string: "| 01 | 認識資料庫 | ✅ | - | - | 2024-12-16 |"
```

### `/grader` 更新範例

```markdown
# 更新多個欄位
1. 更新測驗狀態與成績
old_string: "| 01 | 認識資料庫 | ✅ | - | - | 2024-12-16 |"
new_string: "| 01 | 認識資料庫 | ✅ | ✅ | 85% | 2024-12-16 |"

2. 更新平均成績（若需要）
old_string: "| 平均成績 | 0% |"
new_string: "| 平均成績 | 85% |"
```

---

## 成績判定標準

- **通過（✅）**: 成績 >= 80%
- **需複習（🔄）**: 成績 < 80%
- **未測驗（-）**: 尚未進行測驗

---

## 複習建議

當成績 < 80% 時，自動加入「需複習章節」清單：

```markdown
## 需複習章節

1. 第 XX 章：章節名稱（成績 XX%，建議於 YYYY-MM-DD 後重測）
```

**複習日期計算**：批改日期 + 3 天
